Daily Progress

For my miniterm project I chose to pursure a more in depth exploration of web development, using tools such as jQuery, Bootstrap, and PHP. Throughout this mintierm, I learned many useful and valuable lessons. The outline below details my schedule, so that you too may become a better web developer.


Day 1: SEO, UX, Mobile-Friendly

On day 1, I sought to explore the basic concepts behind the web interface and how we interact with websites. I explored search engine optimization, the ux/ui, and mobile-friendliness. Search engine optimization (seo), is defined as "the process of affecting the visibility of a website or a web page in a web search". It is the process by which a search engine such as google determines the 'ranking' of a page on its list of results. The user experience and user interface is another similarly important facet of web development. These concepts can be used to determine the optimum design and layout of a page to facilitate ease of use and ease of access. Lastly, as mobile devices are becoming increasingly common, it is important to ensure that your webpage is mobile friendly. It would be a poor user experience to have tiny buttons and tiny text on a 4 inch screen, so the way we solve this issue is by using modals. Modals can control the css that is active based on the screen size detected, allowing you to effectively scale your page for smaller devices.


Day 2: Javascript and jQuery

Javascript is a powerful tool to use in order to create responsive web pages. Javascript allows you to control the behavior of a webpage, based on the actions of a user. For example, you could use the javascript onclick=function() event to set a function that excecutes whenever a user clicks on an element.

jQuery aims to take the javascript one step further, by simplifying common functions. While it may initially seem different from javascript, jQuery is infact simply a javascript library. It makes things like HTML document traversal and manipulation, event handling, animation, and Ajax much simpler with an easy-to-use API that works across a multitude of browsers.


Day 3: HTML/CSS and Bootstrap

Together, HTML and CSS make up the backbone of nearly every website on the internet. HTML, or hypertext markup language, controls the structure and shape of a webpage, by defining elements such as headers h1, divs div, and lists ul. CSS works in conjunction with HTML, by delivering an effective method to stylize and customize the html elements. It allows you to set properties such as background-color, width, and margin.

Bootstrap, unlike jQuery, is not a library for HTML, it is a framework which includes countless defined HTML elements and their corresponding styling. It serves to speed up web development by providing many of the basic elements needed and by implementing mobile-friendly development. In addition to the common html elements, it also includes numerous custom HTML and CSS components and jQuery plugins.


Day 4: PHP

PHP is a recursive acronym for PHP: Hypertext Preprocessor. PHP is a widely used scripting language which can be embedded into HTML and used to carry out a variety of tasks. The embedded code from PHP is executed on the server, then the results are sent to the web browser. Since the browser only receives the results of the PHP execution, and not the code, it can become an incredibly versatile tool. Of note is the function include. The include function can be used to include other html files within the main html file itself. The browser is unable to tell the difference between the two, since it is only given the final output, and not the code beneath it.


Day 5: Building a Webpage

Now, it is time to put together all the HTML, CSS, and JS. Using Bootstrap for some HTML and CSS components and jQuery for some Javascript components, I was able to finish the majority of the webpage. All that remains is to make edits and ensure that everything runs as smoothly as intended.

When we are in the progress of building our site, it is helpful to open the index.html file in a web browser, so that we can see the changes which are being made.


Day 6: Polishing and Refining

Before we publish our webpage, it is important to make sure that we have made any changes and polished it so that it remains uncluttered and pleasant to use. In some instances, elements are not scaling properly with reduced screen sizes, so we must add additional modals dictating their behavior and styling.

After making final changes, we can push our files to the NCSSM cs2 server using scp. In my case, I used scp -r * li18s@cs2.ncssm.edu:~/public_html/, but you will need to use your own login info. The -r specifies recursion, so that folders are copied as well. The * specifies that everything that is in the current directory is to be copied.
